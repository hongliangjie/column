## What this repo is

A Jekyll-based personal tech column (Chinese language) published at http://column.hongliangjie.com. Content covers AI/ML, team management, and industry analysis. Posts live in `_posts/` as Markdown files; images live in `assets/`.

## Commands

```bash
./run.sh          # Local dev server (sets macOS SDK env vars, then: bundle exec jekyll serve)
./test.sh         # Dev server that also renders _drafts/
./build.sh        # Production build → _site/ (sets same macOS SDK env vars)
```

The SDK env vars (`SDKROOT`, `LIBRARY_PATH`) in `run.sh` and `build.sh` are required on macOS for native gem compilation. Do not remove them.

## Creating a new post

1. Add a Markdown file to `_posts/` named `YYYY-MM-DD-slug.md`.
2. Required front matter:
   ```yaml
   ---
   layout: post
   title: 文章标题
   categories: 管理
   ---
   ```
   - `categories` is a comma-separated string (e.g., `读书笔记,管理`). No YAML list syntax.
3. Images go in `assets/` and are referenced as `![](/assets/filename.jpg)`.

## Post Markdown style conventions

- Opening paragraph before any heading — no heading for the intro.
- Section headings: `## Title` followed immediately by `***` on the next line, then optionally `![](/assets/img.jpg)`.
- Sub-sections use `###`.
- Horizontal `---` rules are used sparingly as narrative separators within a section (not between sections).

## Image workflow

- Target file size: **100–200 KB**, max dimension **800 px** on the longest edge.
- Resize with `sips` (macOS built-in): `sips -Z 800 --setProperty formatOptions 78 file.jpg`
- Convert PNG → JPEG when it produces a smaller result: `sips -s format jpeg --setProperty formatOptions 80 input.png --out output.jpg`
- Source images from Wikimedia Commons (freely licensed). Download with `curl -L`.
- Prefer landscape (wider than tall) photos; avoid portrait-oriented images.

## Architecture

| Path | Purpose |
|---|---|
| `_posts/` | All published articles; filename encodes publish date |
| `_drafts/` | Unpublished drafts (rendered by `./test.sh`) |
| `assets/` | All images; no subdirectory structure |
| `_layouts/` | `default.html`, `page.html`, `post.html` |
| `_includes/` | `header.html`, `footer.html`, `sidebar.html`, social icon partials |
| `_sass/` | `_base.scss`, `_layout.scss`, `_syntax-highlighting.scss` |
| `_site/` | Build output — not committed, regenerated by `./build.sh` |
| `_config.yml` | Site title, URL, pagination (10/page), `jekyll-sitemap` plugin |

Jekyll version is pinned via `github-pages` gem (v232, Jekyll 3.10.0) for GitHub Pages compatibility.
